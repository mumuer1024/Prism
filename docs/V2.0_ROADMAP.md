## 一、产品定位

### 1.1 核心定位

| 项目 | 说明 |
|------|------|
| **产品性质** | AI 驱动的开源情报聚合工具 |
| **商业模式** | 工具功能订阅（使用次数制） |
| **AI 调用** | 用户自备 API Key，直连模型服务商 |
| **数据流向** | 用户 → 棱镜工具 → 用户选择的 AI 服务商 |
| **棱镜的角色** | 不经手、不存储、不代发 API 请求 |

### 1.2 核心原则

```
✅ 用户自备 API Key，存储在浏览器本地
✅ 我们不触碰用户的 API Key 和 AI 请求内容
✅ 不提供在线支付，通过激活码充值使用次数
✅ 用户数据 30 天自动清理，不做长期存储
✅ 邀请返利仅限首次消费，简化逻辑
```

### 1.3 服务分层

| 版本 | 使用次数 | 功能限制 | 价格 |
|------|----------|----------|------|
| **免费版** | 每日 3 次 | 基础数据源 | 免费 |
| **专业版** | 按激活码充值 | 全部数据源 + 自定义 + 预设广场 | 按次数收费 |
| **团队版** | 按激活码充值 | 专业版 + 多席位 + 协作功能 | 按次数收费 |

## 六、核心业务逻辑

### 6.1 使用次数扣减逻辑

```python
def deduct_usage(user_id: int, tool_type: str) -> dict:
    """
    扣减使用次数
    优先级：付费次数 > 免费次数
    """
    user = get_user(user_id)
    today = datetime.now().strftime('%Y-%m-%d')
    
    # 重置每日免费额度
    if user.daily_free_date != today:
        user.daily_free_used = 0
        user.daily_free_date = today
    
    DAILY_FREE_LIMIT = 3
    
    # 情况1：有付费次数，优先使用
    if user.usage_count > 0:
        user.usage_count -= 1
        save_user(user)
        return {"success": True, "source": "paid", "remaining": user.usage_count}
    
    # 情况2：无付费次数，检查免费额度
    if user.daily_free_used < DAILY_FREE_LIMIT:
        user.daily_free_used += 1
        save_user(user)
        remaining = DAILY_FREE_LIMIT - user.daily_free_used
        return {"success": True, "source": "free", "remaining": remaining}
    
    # 情况3：无可用次数
    return {"success": False, "message": "使用次数不足，请充值"}
```

### 6.2 激活码兑换逻辑

```python
def redeem_activation_code(user_id: int, code: str) -> dict:
    """
    兑换激活码
    """
    # 1. 验证激活码
    activation = get_activation_code(code)
    if not activation:
        return {"success": False, "message": "激活码不存在"}
    if activation.used:
        return {"success": False, "message": "激活码已使用"}
    
    # 2. 获取用户信息
    user = get_user(user_id)
    
    # 3. 计算实际充值次数
    actual_count = activation.usage_count
    
    # 4. 检查邀请奖励（仅首次消费）
    inviter_reward = 0
    if not user.has_purchased and user.invited_by:
        inviter = get_user(user.invited_by)
        if inviter:
            # 邀请人获得奖励
            inviter_reward = 5  # 或按比例计算
            inviter.usage_count += inviter_reward
            save_user(inviter)
            
            # 被邀请人获得折扣或额外次数
            actual_count += 2  # 被邀请人额外赠送次数
            
            # 记录奖励
            create_referral_reward(
                inviter_id=user.invited_by,
                invitee_id=user_id,
                order_code=code,
                reward_type="extra_count",
                reward_value=inviter_reward
            )
    
    # 5. 充值
    user.usage_count += actual_count
    user.has_purchased = True
    save_user(user)
    
    # 6. 标记激活码已使用
    activation.used = True
    activation.used_by = user_id
    activation.used_at = datetime.now()
    save_activation_code(activation)
    
    return {
        "success": True,
        "usage_count_added": actual_count,
        "total_usage_count": user.usage_count,
        "inviter_reward": inviter_reward if inviter_reward > 0 else None
    }
```

```python
def cleanup_expired_reports():
    """
    清理过期报告（定时任务，每日执行）
    """
    now = datetime.now()
    expired_reports = get_reports_where(expires_at__lt=now)
    
    for report in expired_reports:
        delete_report(report.id)
    
    logger.info(f"Cleaned up {len(expired_reports)} expired reports")
```

## 十、风险与缓解

| 风险 | 级别 | 缓解方案 |
|------|------|----------|
| 激活码被滥用 | 中 | 绑定使用设备、限制单码使用次数 |
| OAuth 国内访问不稳定 | 中 | 提供邮箱注册作为备选 |
| 邮件验证码被拦截 | 中 | 提供多种验证方式、检查垃圾邮件提示 |
| 免费用户数据存储限制 | 低 | 提示定期导出、付费版提供云端存储 |
| SQLite 并发限制 | 低 | 单机部署足够，未来可迁移 PostgreSQL |
